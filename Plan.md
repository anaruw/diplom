1. Сервис платежей
   - сценарий с вводом номера карты со статусом "APPROVED"
   - сценарий с вводом номера карты со статусом "DECLINED"

# Перечень используемых инструментов

1. IDE: IntelliJ IDEA (Community Edition)
   - знакома и готова к работе

2. Автоматизация сборки проекта: Gradle
   - немногословный файл настройки, по сравнению с Maven
   - лучше изучены некоторые аспекты настройки
  
3. Инструмент для работы с контейнерами: Docker
   - аналоги не изучены, контейнеры нужны для СУБД и приложения
  
4. Инструмент для работы с БД: DBeaver CE
   - проще и быстрее работать в знакомой программе с графическим интерфейсом, нежели в консоли

5. Контроль версий: Git
   - изучена, достаточна для работы
  
6. Браузер: Google Chrome
   - для взаимодействия с веб интерфейсом приложения(требований по совместимости с браузерами нет, взял, что под руку попалось)

7. Фреймворки:
   1. Project Lombok
      - сокращает объем шаблонного кода с помощью аннотаций
   3. Selenide
      - код проще, чем в Selenium
      - минимум необходимых настроек
   5. JUnit-jupiter
      - современная библиотека для написания тестов
   7. Javafaker
      - для создания случайных данных
   9. Allure
       - отчетность по результатам тестов информативнее, чем в Gradle
